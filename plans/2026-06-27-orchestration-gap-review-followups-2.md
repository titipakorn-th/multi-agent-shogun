## P0-1: Fix weekly health review validation-miss logic

Delegate to: fixer, then oracle review

Problem:
`scripts/weekly_health_review.sh` says it checks "cmds without
required_validations", but the current implementation increments the counter
when a `status: done` command has `required_validations` present. The tests now
lock in the inverted behavior.

Files:
- `scripts/weekly_health_review.sh`
- `tests/unit/test_weekly_health_review.bats`

Acceptance criteria:
- A done command with no `required_validations` is counted as a validation miss.
- A done command with non-empty `required_validations` is not counted as a miss.
- Pending/in-progress commands with or without `required_validations` are not
  counted as misses.
- Both archive shapes are covered:
  - `queue/shogun_to_orchestrator_archive.yaml`
  - `queue/archive/*.yaml`
- Tests are renamed away from "done+required_validations increments counter" and
  assert the corrected behavior.
- `bats tests/unit/test_weekly_health_review.bats` passes.
